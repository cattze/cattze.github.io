Cookie、 Web Storage

发表于 2018-03-24 | 分类于 HTML

Cookie
主要用途有保存登录信息，比如你登录某个网站市场可以看到“记住密码”，这通常就是通过在 Cookie 中存入一段辨别用户身份的数据来实现的。
Web Storage
HTML5 提供了两种在客户端存储数据的新方法：
localStorage - 没有时间限制的数据存储，第二天、第二周或下一年之后，数据依然可用。
sessionStorage - 针对一个 session 的数据存储， 当用户关闭浏览器窗口后，数据会被删除。

特性 Cookie localStorage sessionStorage
数据的生命期 一般由服务器生成，可设置失效时间。如果在浏览器端生成Cookie，默认是关闭浏览器后失效 除非被清除，否则永久保存 仅在当前会话下有效，关闭页面或浏览器后被清除
存放数据大小 4K左右 一般为5MB 一般为5MB
与服务器端通信 每次都会携带在HTTP头中，如果使用cookie保存过多数据会带来性能问题 仅在客户端（即浏览器）中保存，不参与和服务器的通信 仅在客户端（即浏览器）中保存，不参与和服务器的通信
易用性 需要程序员自己封装，源生的Cookie接口不友好 源生接口可以接受，亦可再次封装来对Object和Array有更好的支持 源生接口可以接受，亦可再次封装来对Object和Array有更好的支持

localStorage 和 sessionStorage 操作
1）保存: setItem存储value，将value存储到key字段

sessionStorage.setItem(“key”, “value”) or sessionStorage.key=”value”;
localStorage.setItem(“lastname”, “Smith”) or localStorage.lastname=”Smith”;

2）读取: getItem获取value，读取指定key本地存储的值
var value = sessionStorage.getItem(“key”) or var value = sessionStorage.key; 
var name = localStorage.getItem(“lastname”) or var name = localStorage.lastname;

3）删除: removeItem删除key， 删除指定key本地存储的值
sessionStorage.removeItem(“key”); 
localStorage.removeItem(“lastname”);

4）清除: clear清除所有的key/value
sessionStorage.clear(); 
localStorage.clear();
