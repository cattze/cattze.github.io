React Router

发表于 2017-08-26 | 分类于 React

React Router 是完整的 React 路由解决方案。
React Router 保持 UI 与 URL 同步。它拥有简单的 API 与强大的功能例如代码缓冲加载、动态路由匹配、以及建立正确的位置过渡处理。
通过 npm 安装：


  $ npm install –save react-router

Router组件本身只是一个容器，真正的路由要通过Route组件定义。


  import { Router, Route, hashHistory } from ‘react-router’;
  React.render((
  <Router history={hashHistory}>
    <Route path=”/“ component={App}>
      <Route path=”about” component={About} />
      <Route path=”inbox” component={Inbox} />
    </Route>
  </Router>
  ), document.body)

URL组件
/App
/about App -> About
/inbox App -> Inbox

最外层组件是 Router（可以把他看作是react-router提供的最外层容器） , 下一层是 Route，这个是路由组件。
Router组件有一个参数history，它的值hashHistory表示，路由的切换由URL的hash变化决定，即URL的#部分发生变化。如，用户访问 http://www.XXXXXX.com/，实际会看到的是 http://www.XXXXXX.com/#/。
path 对应路径，component 对应当前路径渲染的组件。
Route 里面的 Route 表示在父组件路由的 path 路径下面的一级 path 对应的路由，这里的路由是父子嵌套，对应的组件也同样是父子嵌套的。
也可以直接使用原生 route 数组对象。单独传入 Router 组件的 routes 属性。


  const router = [{
    path: ‘/‘,
    component: App,
    childrenRoutes: [
      { path: ‘about’, component: About },
      { path: ‘inbox’, component: Inbox },
    ]
  }];
  render((<Router routes={router} />), document.getElementById(‘root’));

上面代码中，用户访问/about时，会先加载App组件，然后在它的内部再加载About组件。
App组件要写成如下，App组件的this.props.children属性就是子组件。


  export default React.createClass({
    render() {
      return 

        {this.props.children}
      
    }
  })

想象一下当 URL 为 / 时，我们想渲染一个在 App 中的组件。不过在此时，App 的 render 中的 this.props.children 还是 undefined。这种情况我们可以使用 IndexRoute 来设置一个默认页面


  <Router history={hashHistory}>
    <Route path=”/“ component={App}>
      { 当 url 为 / 时渲染 Dashboard }
      <IndexRoute component={Dashboard} />
      <Route path=”about” component={About} />
      <Route path=”inbox” component={Inbox} />
    </Route>
  </Router>

注意，IndexRoute组件没有路径参数path。
